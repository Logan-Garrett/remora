name: test-plan
description: Generate a test plan for the changes discussed in this session
---

Based on what has been discussed and changed in this session, create a test plan.

For each change, list:
1. **What to test** — the specific behavior or feature
2. **How to test it** — concrete steps (manual or automated)
3. **Edge cases** — boundary conditions, error paths, concurrency concerns
4. **What could break** — other parts of the system that might be affected

If automated tests should be written, suggest the test file location and key test cases (function names + one-line descriptions).
