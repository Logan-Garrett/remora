name: explain
description: Explain the current codebase or a specific file to a new team member
---

Explain the code that has been discussed or shared in this session. Assume the reader is a developer joining the team who hasn't seen this code before.

Cover:
- What this code does at a high level
- Key data structures and their relationships
- The flow of a typical request/operation through this code
- Any non-obvious design decisions and why they were made

Use clear language. Avoid jargon unless you define it.
